# The Job Shop Scheduling Problem (JSSP)

One common scheduling problem is the job shop scheduling problem. There are multiple
versions of this problem but we are only concerned about the one in which multiple jobs are
processed on a single machine. For each job the duration and the due date is known. We
need to find the sequence of jobs that minimizes the total delay of all the jobs. This problem
is an NP-Hard problem and has n! solutions. So Local Search seems to be a great way to
find an optimal solution.
There are several constraints for the job shop problem:

● No task for a job can be started until the previous task for that job is completed.

● A machine can only work on one task at a time.

● A task, once started, must run to completion (no parallel task allowed)

In this assignment you will be solving JSSP using local search.

# Simulated Annealing

You can find the details about Simulated Annealing in your textbook. However, the provided
pseudocode in the book is for maximization problems, while JSSP is a minimization problem.
In this assignment you will be implementing it in the following manner.

current = randomly generated initial state

for (t = 1; ; t++)

T = schedule (t)

if (T == 0) return current

ran = random number between [0,1]

if (ran < 0.5)

next <- a successor of current generated by “heuristic1”

else

next <- a successor of current generated by “heuristic2”

delE = cost(next)-cost(current)

if next better than current i.e., delE < 0

current = next

else

current = next only with probability exp(-delE/kT)

# Scheduling function

The scheduling function (schedule) should provide larger T at the beginning and gradually
decrease it to 0. If T is lowered slowly enough, Simulated Annealing will find a global
optimal solution with probability approaching 1. However, finding such function is very hard
in practice.
You can find many scheduling functions in the literature. Some are given below:

### Linear cooling scheme: schedule (t) = T0 - ηt

### Exponential cooling scheme: schedule (t) = T0αt, here 0<α<1

### Logarithmic cooling scheme: schedule (t) = c / log (1+t)

### In this assignment you will be implementing the Exponential cooling scheme.

# Heuristics for generating neighbors

In literature one can find many well-known heuristics to generate neighbors of JSSP. Some
of them are greedy and some are random. You will implement the following heuristic and
find a new heuristic by yourself in this assignment.

## 1-1 Exchange
Randomly pick two jobs from the sequence and exchange them. A pseudocode is given
below.

### oneOneExchange(jobsequence)

{

n = length of jobsequence;

ran = random integer between [1, n]

ran2 = random integer between [1, n] and not equal to ran

swap the jobs at ran and ran2

return jobsequence

}

## THE TASKS

1. Implement Simulated Annealing with the heuristic specified above. Figure out at least
one more heuristic by yourself and if possible more than one. At the time of generating
neighbor, select one of the two heuristics uniformly, i.e., with 50% chance Heuristic 1
will be used to generate a neighbor and the rest of the time Heuristic 2 will be used.

2. Write a code to make at least 10 valid and random dataset with at least 500 jobs for this
problem. Submit the dataset along with the zipped file.

3. Using the datasets you have created, run your code with different values of the
parameter T0, α, and k. Each time you will run the algorithm 10 times, compute some
statistics and present them as shown in the table. You must prepare the result before
submission and show it in hard copy at the time of evaluation. Make sure the dataset
you have given in the zip file is used for generating this result.
